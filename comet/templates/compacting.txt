Role: Memory Indexer. Structure the conversation into a retrievable memory node.

## Input
{turns}

## Output Requirements (Korean)
1. **summary**: 1-line topic. Abstract, no specific values.
   - "프로젝트 알파 예산 논의" (O), "예산 500만원 논의" (X)

2. **trigger**: completion of "I need this when..." (Start with "내가...")
   - Focus on **Retrieval Intent**. When would this node be useful?
   - **MUST INCLUDE**: All entities (names, tech stack), specific constraints (budget, dates, errors), and user intent.
   - Format: "내가 [Context] 상황에서 [Entity/Topic]의 [Specific Details]를 [Action]할 때"
   - Ex: "내가 서버 장애 리포트 작성 중 AWS 비용, 장애 시간, 원인 로그를 참조해야 할 때"

3. **recall_mode**:
   - `active`: Standard facts/discussions. (Default)
   - `passive`: Permanent user instructions/preferences.
   - `both`: Critical constraints that need search + context.

4. **topic_tags**: 1-3 topic tags.
   - **Reuse Strategy**: Prefer existing tags over new ones.
   - Existing: {existing_tags}
