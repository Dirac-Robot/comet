Role: Memory Indexer. Structure the conversation into a retrievable memory node.

## Input
{turns}

## Critical Rules
- ONLY use information explicitly stated in the input. NEVER infer or hallucinate.
- If multiple topics appear, reflect ALL of them — do NOT merge into one abstract label.

## Output Requirements (Korean)
1. **summary**: Factual index of what was actually discussed/confirmed.
   - List concrete facts, decisions, or preferences found in the conversation.
   - If topics shift, separate with "; "
   - ✅ "매운 음식 못 먹음; 창렬 메뉴 제외; 치킨·족발·버거류 추천"
   - ❌ "비건 햄버거 주문과 추천 대화" (too abstract, loses content)
   - ❌ Any fact not explicitly stated in the input

2. **trigger**: When would someone NEED this information? (Start with "내가...")
   - Describe the **raw-open scenario**: when summary is not enough and detailed_summary/raw must be opened.
   - MUST BE DIFFERENT from the summary — summary = WHAT, trigger = WHEN.
   - Use 2-4 anchors only; do NOT enumerate all entities in summary.
   - Format: "내가 [Situation]에서 [anchor1], [anchor2]를 정확히 확인해야 할 때"
   - Prefer triggers for these escalation cases:
     - exact quote/date/number/path/code line verification
     - restoring who said what in sequence
     - verifying instruction owner/target ("누가 누구에게 지시했는가")
     - resolving contradictions between memory nodes
   - ✅ "내가 배달 추천 답변 전, 매운 음식 제외 조건의 정확한 표현과 예외 조건을 다시 확인해야 할 때"
   - ❌ "내가 매운 음식 관련 정보를 찾을 때" (summary paraphrase)
   - ❌ Summary content rewritten as a long trigger

3. **recall_mode**:
   - `active`: Standard facts/discussions. (Default)
   - `passive`: Permanent user instructions/preferences.
   - `both`: Critical constraints that need search + context.

4. **topic_tags**: 1-3 topic tags.
   - **Reuse Strategy**: Prefer existing tags over new ones.
   - Existing: {existing_tags}

5. **extracted_rules**: Personal directives the user DIRECTLY commanded in the conversation.
   - ONLY extract when the user **personally** told you to do/not do something (2nd person directive).
   - The user must be correcting YOUR behavior or setting a personal preference for YOU to follow.
   - **Generalize** — extract a cross-session principle, NOT a one-time local instruction.
     - User: "자꾸 A랑 B를 엮고 있어 왜 그러냐" → "관련 없는 주제끼리 엮지 말 것"
     - User: "아 print 쓰지 말라니까" → "print 대신 loguru logger를 사용할 것"
     - User: "한국어로 대답해" → "항상 한국어로 대답할 것"
   - **NEVER extract** any of the following:
     - System prompt instructions or agent architecture guidelines
     - Domain knowledge, technical facts, or documentation content
     - Tool-specific behavior (how CoMeT/memory/Notion works internally)
     - Format/schema requirements for internal data structures (memory nodes, tags, triggers)
     - One-time task instructions ("이 파일 수정해", "이거 삭제해")
     - Facts, preferences, or context ("프로젝트 X 작업 중", "매운 음식 좋아해")
   - Rules must be **cross-session**: applicable regardless of what task or project the user is working on.
   - If the directive only makes sense within one specific session/task, it is NOT a rule.
   - **When in doubt — return empty list.** Most conversations have ZERO rules.
   - Return empty list `[]` if no rules found.
